MOVIE RECOMMENDATION SYSTEM


Different people have preferences for a variety of products where each person has an affinity towards a specific product.This affinity between users and products is measured using user-product relationship. Some products are similar in nature such as books of the same genre, dishes from the same cuisine and this extent of similarity is measured using product-product relationship.Some people are similar is nature because of their age, gender,background and the similarity is measured using user-user relationship.Based on these relationships, we can provide tremendous insights such as recommending similar movies to a specific user (user-product relationship), suggesting an ear phones for a phone(product-product relationship) and suggesting friends to friends on facebook(user-user relationship).Insights can be derived from relationships which can be monetized in the form of recommendations.By knowing the right form of recommendations, we can send personalized promotional emails to the user and can create a personalized home page whenever user visits a web site.

MOTIVATION

Recommender system is very useful for personalization, reducing organizational maintenance, always keeping us up-to-date and suggesting based on real activity. In spite of the emergence of various new forms of entertainment, movies have always been the most prominent and one of the highly preferred forms of entertainment. Everybody loves customization and viewers only want to spend their time and money on something that is close to their taste and something that would get a return on their investment. So, the movie recommender system provides highest possibility for people to like the recommended movies instead of spending their valuable time on searching and watching movies which they wouldn’t like much.



ALGORITHMS IMPLEMENTED


K NEAREST NEIGHBORS

The rating for a movie by a user is estimated by taking the average of ratings of similar users who have already rated that movie.
  1. Estimate the rating for unrated movies.
    • Pick k-nearest users for our user and then take the average of the ratings for unrated movies.The average may be a simple average or a weighted average which considers the extent of similarity of each user with our user.
    • Here we used the simple average of ratings given by similar users.
    • The similarity between two users can be measured by taking movies which are seen to both users and comparing the ratings given by both users. If the ratings given by both users on the common movies are almost same, then the two users are very similar.
    • If the users are represented as points in an N-dimensional plane, then the similarity is measured by the distance between them.
    • The distance between two users is measured by hamming distance.It computes how many numbers agree between the two sets of ratings given by both users.
  2. Sort them in descending order.
  3. Pick the top ten movies for recommendation.

CONTENT BASED FILTERING

In content based filtering, we will find movies with similar attributes and suggest it to the user. For example, we can suggest the movie “B” to the user who liked movie “A” if both the movies have similar rating against a few same attributes as direction, cast, genre (product attribute similarity).
  1. Content based filtering requires movie attribute database of the user for whom we would like to recommend movies.
  2. The attributes are to be picked based on their impact on the user preferences.
  3. Each movie is shown as a point in an N-dimensional space and the similarity is measured based on the distance between the two points.
  4. Movies are recommended based on their distance from the rated movies by the user.

NAIVE BAYES

Naive Bayes classifier is a powerful and straightforward algorithm for the classification task. This classifier gives great results when used for textual data analysis. The naive bayes classifier technique is a conditional probability model based on Bayesian theorem with strong independence assumptions between the features.

CHALLENGES

K Nearest Neighbors:
  • K value cannot be choosen randomly.The accuracy of the recommendation system depends on the K value hence it should be choosen carefully.As tested,a large K value is more precise as it reduces the overall noise, k value is choosen as 50.
  • There are differnt metrics to calculate distance between two users to know the similarity between them.The different metrics are euclidean,correlation and hamming distance.Choosing the distance metric is also a challenge as the accuracy changes based on the distance metrics.We calculated the distance using all the distance metrics and found that euclidean distance gives more accurate results.

IMPLEMENTATION

K Nearest Neighbors

  1. Read the user,movies and ratings data.
  2. Calculate the number of users and movies to build a matrix.
  3. Populate the matrix(rows represent users and columns represent movies) with rating values.
  4. Construct a distance matrix which contains distances from each user to all other users.The distance is calculated using euclidean distance function(euclidean fn is imported from scipy.spatial.distance library).
  5. Calculate the top 100 users similar to the user for whom movies are being recommended.
  6. Calculate the mean movie scores for all the movies rated by the top 100 similar users.
  7. Sort the mean movie scores and take their movie indexes.
  8. Remove the movies seen by the current user from the sorted set.
  9. Print top 20 movies recommended to the current user.

NAIVE BAYES

  1. Read the training,test,movie data.
  2. Group the training data based on user and separate the user_ids,movie_ids,ratings.The same step is done with the testing data.
  3. Find the maximum user and movie indices in training data and build a matrix of that size and populate the ratings in it.
  4. The ratings are populated in the matrix such that any rating>=3 is given a value of 1 and any rating<=3 is given -1
  5. Calculate the total number of ratings by counting the number of ones and minus ones in the matrix and adding them.
  6. A function naive_bayes_classifier is defined where a matrix containing probabilities for the each position of the rating matrix is built.
  7. Compare the predicted probability matrix to the test probability matrix to calculate accuracy.
  8. Recommend top 25 movies.
  9. Run the job using the following command spark-submit naivebayes_scratch.py <.base filepath> <.test filepath> <.item filepath>

Content Based

  1. Choose the genres from the top rated movies of the user
  2. Average of the ratings given by other users is calculated for each movie of that genre.
  3. Top 20 movies of that genre are recommended to that user.
